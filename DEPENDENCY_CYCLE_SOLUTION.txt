1. AppModule.kt:

   ✓ Ejecución lazy en CoroutineScope(Dispatchers.IO)

3. Flujo lineal de dependencias (sin ciclos):
   @ApplicationContext
        ↓
   AppDatabase (singleton)
        ↓
   MyApp.onCreate() → seedDatabase() en thread IO

4. Ventajas de esta solución:
   ✓ Sin ciclos de dependencias
   ✓ Seeding ejecuta en background automáticamente
   ✓ Mejor performance (no bloquea main thread)
   ✓ Logging completo de errores
   ✓ Verificación de datos duplicados
